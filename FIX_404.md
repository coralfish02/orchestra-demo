# 🔧 404エラーの対処法

## 考えられる原因

1. **サーバーが起動していない**
2. **ビルドエラーがある**
3. **コンポーネントのインポートエラー**
4. **ポートが間違っている**

## 対処法

### ステップ1: サーバーを再起動

```powershell
cd "C:\Users\hp\OneDrive - Sophia Univ. Students\ドキュメント\googlehackthon2026\demo\orchestra-practice-app"

# サーバーを停止（Ctrl+C）
# 再度起動
npm run dev
```

### ステップ2: ビルドエラーを確認

```powershell
npm run build
```

エラーが出た場合は、そのエラーメッセージを確認してください。

### ステップ3: 依存関係を再インストール

```powershell
rm -rf node_modules package-lock.json
npm install
npm run dev
```

### ステップ4: 正しいURLにアクセス

- ✅ 正しい: `http://localhost:3000`
- ❌ 間違い: `http://localhost:3000/` (末尾のスラッシュは問題ないですが、ルートパスを確認)
- ❌ 間違い: `http://127.0.0.1:3000` (localhostを使う)

### ステップ5: ブラウザのキャッシュをクリア

1. ブラウザで Ctrl+Shift+Delete
2. キャッシュをクリア
3. ページを再読み込み（Ctrl+F5）

### ステップ6: 別のポートで試す

```powershell
npm run dev -- -p 3001
```

ブラウザで `http://localhost:3001` を開いてください。

## よくあるエラー

### エラー: "Cannot find module '@/components/...'"

**対処法**: `tsconfig.json` のパス設定を確認

```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```

### エラー: "Module not found: Can't resolve '@/data/beethoven_5_data.json'"

**対処法**: `tsconfig.json` に以下を追加

```json
{
  "compilerOptions": {
    "resolveJsonModule": true
  }
}
```

### エラー: ビルドは成功するが404エラー

**対処法**: 
1. `.next` フォルダを削除
2. 再ビルド

```powershell
rm -rf .next
npm run build
npm run dev
```

## 確認チェックリスト

- [ ] サーバーが起動している（"Ready" と表示されている）
- [ ] ビルドエラーがない
- [ ] 正しいURL（http://localhost:3000）にアクセスしている
- [ ] ブラウザのコンソール（F12）でエラーを確認
- [ ] PowerShellの出力でエラーメッセージを確認

## デバッグ方法

### ブラウザのコンソールを確認

1. F12を押す
2. 「Console」タブを開く
3. エラーメッセージを確認

### PowerShellの出力を確認

開発サーバーを起動しているPowerShellウィンドウでエラーメッセージを確認

### ネットワークタブを確認

1. F12を押す
2. 「Network」タブを開く
3. ページを再読み込み
4. 404エラーが出ているリソースを確認

## それでも解決しない場合

1. **すべてのNode.jsプロセスを終了**：

```powershell
taskkill /F /IM node.exe
```

2. **再度起動**：

```powershell
npm run dev
```

3. **エラーメッセージを共有**：
   - PowerShellの出力
   - ブラウザのコンソールのエラー
   - ビルドエラー（あれば）
